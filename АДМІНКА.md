# Адмінка BUDZIRKA — коротка інструкція

**Адреса:** https://budzirka-configurator.vercel.app/admin

## Вхід

Введи свій **ключ доступу** (видається окремо, у цей документ не пишемо).
Ролі:
- **👑 Адміністратор** — усе, включно з «Налаштуваннями» цін і видаленням декорів.
- **👤 Менеджер** — усе, крім «Налаштувань» і видалення.

Ключі зберігаються в Supabase → таблиця `budzirka_admins` (там їх можна змінити чи додати нові).

## Головне правило збереження

- Змінив щось у рядку → рядок підсвічується **жовтим** = ще не збережено.
- Тисни **«Зберегти»** в рядку, або **«Зберегти всі»** на чорній панелі знизу (з'являється, коли є зміни).
- Якщо спробуєш піти зі сторінки з незбереженим — система попередить.

## Вкладки

### 📋 Заявки (CRM)
Усі ліди з конфігуратора, нові — зверху. Червоний лічильник на вкладці = кількість нових.
- **Статус**: Новий → В роботі → КП надіслано → Угода / Відмова — обери і збережи.
- **Нотатка** — домовленості, наступний крок.
- **📞 кнопка-телефон** — дзвінок в один тап (якщо клієнт просив «не дзвонити» — стоїть 📵, кнопки немає).
- «Склад КП» — розгорнути, щоб побачити позиції й суми.
- Зверху: **пошук** (ім'я / телефон / компанія / № КП) і **фільтри за статусом**.

### 📦 Товари
Типи кабін: назва, опис, висота, фурнітура, статус.
- **Фото**: клікни на мініатюру (або 📷) → вибери файл (до 4 МБ) → воно одразу на сайті.
- Перемикач = чи показується тип у конфігураторі.

### 🎨 Декори
Два списки: для кабін і для стільниць.
- **Пошук** за кодом/назвою, фільтр «Немає в наявності».
- **Наявність** (листів) — поряд бейдж «є / немає». 0 = «Під замовлення» на сайті.
- **Ціна €/м²** — для стільниць (порожньо = ціна за замовчуванням з Налаштувань).
- Колір — піпеткою. «＋ Додати декор» — новий рядок (заповни код і збережи).
- Видалення (✕) — тільки адміністратор.

### 🛠 Обробка
Ставки торців №1–9 (€/м.п.) для стільниць.

### ⚙️ Налаштування (тільки адмін)
Глобальні ціни: **ставка кабін €/м²** (зараз 170), курс €→грн, ціна вирізу/отвору, ціна м² за замовчуванням.

## Як зміни потрапляють на сайт

Миттєво: конфігуратор читає каталог із бази при кожному відкритті.
Збережено в адмінці → клієнт, що відкриє сайт, уже бачить нові ціни/фото/наявність.
