# BUDZIRKA · Повноцінний 3D-конфігуратор (v1)

Налаштування виробу **прямо в 3D**: жива сцена + точні контролі + миттєва ціна.
Прев'ю: https://budzirka-configurator.vercel.app/3d-config

## Як працює
- **Постійна 3D-сцена** (Three.js r160) — не перемальовується; при зміні параметра модель **перебудовується наживо**.
- **Контролі** (права панель / нижній лист на мобільному): тип, вид (міжстінна/кутова/окрема), глибина (повзунок+число), довжина фасаду, к-сть дверей стандарт/інклюзив + ширина, декор (свотчі → колір змінюється плавно), фурнітура (пояснення за типом).
- **Перегородки** — автоматично за видом (двері−1+бічні).
- **Жива ціна** € + ≈грн (формула площа×170 €/м², ставка з бази), анімація при зміні.
- **3D-контролі:** автообертання, відкрити двері, показати розміри, скинути вид.
- **Валідація:** глибина 1200–1500, мін. ширина кабінки 1100 — конкретні повідомлення, кнопка блокується.
- **Заявка:** «Сформувати заявку» → модалка (ім'я/телефон/email) → лід у Supabase (`source:'3d-config'`).
- **Каталог із бази** (типи, декори, ставка) з резервним знімком; події аналітики `cfg3d_*`.

## Архітектура (чому так)
Конфігуратор-майстер перемальовує весь DOM щокроку — це знищило б 3D-канвас. Тут навпаки:
**канвас постійний**, оновлюється лише група моделі (`build()`) + панель імперативно. Тому 3D живе й швидке.

## Статус / далі
- v1: кабіни (головний продукт).
- Далі: стільниці в 3D, drag-ручки розмірів просто в сцені, гарячі точки/exploded, AR, інтеграція з основним кошторисом/КП.
- Це **прев'ю-маршрут** — не лінкований з основного сайту, для перегляду й узгодження.
