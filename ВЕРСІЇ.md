# Версії проєкту: як подивитись і як відкотитись

Кожна зміна зберігається **двічі**: як коміт у Git (код) і як деплой на Vercel (живий сайт).
Нічого не зникає — можна подивитись будь-яку стару версію і повернутись на неї за хвилину.

## 1. Подивитись список версій

**Живі версії (кожен деплой має свій URL, всі працюють):**
```
vercel ls budzirka-configurator
```
Відкрий будь-який URL зі списку — побачиш сайт таким, яким він був на той момент.

**Історія коду:**
```
git log --oneline          # повний список змін
git tag -l                 # іменовані версії (v1.0, v1.1…)
```
Або просто на GitHub: https://github.com/Berezhnoy1/budzirka-configurator/commits/main

## 2. Повернутись на стару версію (відкат)

**Швидкий відкат сайту (живе посилання перемикається на старий деплой, ~10 сек):**
```
vercel rollback <URL-старого-деплоя>
```
Приклад: `vercel rollback https://budzirka-configurator-8o2d1yfc0-….vercel.app`
Код у папці при цьому не змінюється — тільки те, що бачать користувачі.

**Повний відкат коду (повертає файли в папці):**
```
git checkout v1.0 -- index.html admin.html    # повернути файли з версії v1.0
git commit -m "rollback to v1.0"
```
Після цього передеплоїти як завжди.

## 3. Іменовані версії (теги)

Великі віхи позначаємо тегами, щоб легко знаходити:
```
git tag -a v1.1 -m "опис версії"
git push origin --tags
```

| Тег | Що в ній |
|---|---|
| v1.0 | Конфігуратор: гейт на головній, кабіни з реальною формулою (170 €/м²), стільниці, кошторис, КП з ескізами, ліди в Supabase, мобільна версія |
| v1.1 | + Адмінка з ролями (адмін/менеджер), каталог у базі, фото товарів, авто-перегородки, CRM-статуси лідів |

## 4. База даних

Зміни в цінах/товарах (через адмінку) живуть у Supabase і в Git не потрапляють.
Snapshot-копія каталогу зашита в `index.html` як резерв. Якщо треба зберегти стан
каталогу перед великими змінами — у Supabase Dashboard: Database → Backups
(щоденні бекапи включені автоматично на 7 днів).
