# BUDZIRKA · 3D-перегляд проєкту

Інтерактивний 3D-переглядач сконфігурованого виробу (сантехнічні кабіни / перегородки).
Відкривається з посилання (у листі або з конфігуратора), показує **саме той виріб**, який зібрав клієнт.

**Live:** https://budzirka-configurator.vercel.app/3d (демо-конфіг без параметрів)

## Що вміє
- Параметрична 3D-модель із конфігурації (тип, розміри, кількість дверей, вид, декор, перегородки, фурнітура).
- Студійне світло + контактна тінь + відображення на хромі — «продуктове фото», як у брендбуку.
- **Орбіта / зум / автообертання**, скидання виду, повний екран.
- **Жива зміна декору** — плавний перехід кольору всіх панелей.
- **Відкриття дверей** — двері обертаються на петлях.
- **Розміри** — винесення габаритів у мм (B2B-довіра).
- **Зберегти фото** (PNG), **AR — у вашому приміщенні** (на телефоні; beta).
- Брендований лоадер, картка специфікації з ціною (€ + ≈грн), CTA «Замовити».
- Аналітика подій у Supabase (`3d_loaded`, `3d_decor`, `3d_doors`, `3d_dims`, `3d_ar`, `3d_cta`).

## Технології
Один файл `index.html`, **Three.js r160** (через CDN, importmap). Без збірки. Модель будується
параметрично з JSON — нічого не треба експортувати, зміни розмірів/декору миттєві.

## Як передається конфігурація
URL: `/3d#cfg=<base64url(JSON)>`. Конфігуратор генерує це посилання (`viewerLink()` в index.html).
Схема JSON:
```json
{ "t":"standart","tn":"MAX Standart","kind":"каб","h":2100,"d":1200,"f":4000,"v":"кут",
  "dc":"0085","dn":"White","hex":"#f1efe9","ns":3,"ws":650,"ni":0,"wi":900,
  "fit":"нержавіюча сталь","core":"black core","price":2284,"cur":"€","sid":"...","tel":"..." }
```
Без `#cfg` показується демо-конфіг.
